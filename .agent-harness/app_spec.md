# AIGC Advertising SaaS Platform

## Product Goal
构建一个面向广告代理商与品牌客户的 AIGC SaaS 平台，支持多租户、多项目、多角色协作，通过统一的模型网关接入市面上通用的大模型平台，以 API 方式提供文生文、文生图、图生图、文生视频、广告文案生成等能力。

平台需要保证：
- 超级管理员可管理租户、模型可用性、平台价格体系和全局安全策略。
- 租户负责人可充值、分配额度、管理成员、项目和品牌素材。
- 普通用户在租户与项目授权范围内使用创作能力。
- 不同模型厂商可替换，平台内部保持统一价格和计量口径。
- 生成内容必须尽量基于品牌自己的素材、知识和规范，避免脱离品牌调性乱生成。

## Primary Users
- 超级管理员：平台运营方，管理模型供应商、模型目录、价格体系、租户启停、全局审计。
- 租户负责人：广告代理商负责人或管理员，充值、配置配额、管理用户和项目。
- 项目管理员：管理某个项目组的成员、品牌、素材库、预算和调用策略。
- 普通用户：创意、文案、设计、投放人员，发起 AIGC 任务并消费额度。
- 品牌客户协作用户：未来可选，用于查看素材和成果，不一定直接调用模型。

## Core Domain Model
- 平台：全局配置、模型供应商、计费规则、审计、风控。
- 租户：一个广告代理商组织，拥有钱包余额、成员、项目、品牌素材。
- 项目：租户下的业务单元，可绑定预算、成员、客户品牌和知识范围。
- 品牌/客户：广告代理商服务的最终客户，拥有品牌介绍、禁用词、风格规则、素材库。
- 模型供应商：如 OpenAI、Anthropic、Google、火山、阿里、Stability、Runway、Pika 等。
- 模型：按能力分类，如聊天、文案、图片、视频、embedding、rerank。
- 价格单位：平台内部统一的计费单位，用于换算不同模型成本。
- 额度策略：按租户、项目、用户限制 token、图片次数、视频秒数、并发数、每日上限。
- 任务：一次 AIGC 调用请求，包含输入、上下文素材、模型选择、结果、消耗、审计信息。
- 素材资产：图片、视频、文档、品牌手册、产品介绍、历史文案、禁用信息。

## Scope For V1
- 多租户与 RBAC：超管、租户负责人、项目管理员、普通用户。
- 多项目：租户下可创建多个项目并分配成员。
- 统一模型网关：通过 API 接入多家模型平台，先不考虑用户自带模型部署。
- 模型能力：文案生成、图片生成、视频生成，后续可扩展更多模态。
- 模型开关：超管决定哪些租户、角色、项目可使用哪些模型。
- 平台价格体系：独立于供应商原价，支持内部统一折算。
- 充值与额度：租户负责人充值，并给用户或项目分配额度。
- 品牌与素材库：每个租户可按品牌/客户沉淀资料和素材，供模型调用时引用。
- 品牌约束生成：生成前注入品牌知识、品牌规则、敏感词限制。
- 基础审计：记录谁在何时用哪个模型生成了什么、花了多少。

## Out Of Scope For V1
- 用户上传私有模型并在平台内托管推理。
- 自建 GPU 集群和自有算力调度。
- 自动投放、媒体采买、广告数据回流闭环。
- 对外客户门户和结算对账单的复杂税务流程。

## Functional Requirements

### 1. Identity, Tenant, and Project
- 支持平台级账号认证与会话管理。
- 支持超管创建、冻结、启用租户。
- 支持租户下多个项目。
- 支持用户在同一租户加入多个项目。
- 支持角色继承与资源级权限控制。

### 2. Model Provider Integration
- 提供统一模型注册表，抽象供应商、模型名、能力类型、计费维度、状态。
- 统一模型调用接口，屏蔽厂商差异。
- 允许同类能力绑定多个候选模型，并支持后续替换供应商。
- 每次调用要记录实际供应商模型和平台计费模型。

### 3. Platform Pricing and Metering
- 平台维护自己的价格体系，不直接暴露供应商原始单价。
- 支持不同维度计费：
  - 文本：输入 token、输出 token。
  - 图片：张数、分辨率档位。
  - 视频：秒数、清晰度档位。
- 支持按模型、能力、租户等级设置不同售价。
- 支持后续修改价格而不影响历史订单账目。

### 4. Recharge, Wallet, and Quotas
- 租户负责人可充值余额。
- 平台从租户余额中扣费。
- 租户负责人可给项目或用户设置额度和限制。
- 额度至少支持：
  - token 上限
  - 图片生成次数
  - 视频生成秒数
  - 每日调用次数
  - 并发任务数

### 5. Brand Knowledge and Asset Library
- 每个租户可维护素材库，并按品牌/客户归档。
- 素材类型包括图片、视频、文档、品牌手册、卖点说明、历史案例。
- 素材可绑定项目和品牌。
- AIGC 任务可引用指定品牌、素材集和知识摘要。
- 平台需要保留可扩展接口，后续可接入 embedding/RAG。

### 6. Creative Generation
- 文案生成：广告标题、描述、脚本、社媒文案等。
- 图片生成：海报、KV、风格图、产品概念图。
- 视频生成：短视频脚本与生成任务。
- 所有生成任务应支持：
  - 选择项目
  - 选择品牌/客户
  - 选择模型
  - 选择或自动注入素材上下文
  - 查看消耗与结果

### 7. Governance and Audit
- 记录任务日志、输入摘要、输出摘要、消耗、费用、调用人、项目、品牌。
- 提供超管和租户管理员可见的审计视图。
- 支持敏感词、禁用品牌词、非法内容策略。
- 关键管理动作需写审计日志。

## Non-Functional Requirements
- 支持多租户数据隔离。
- 模型供应商替换时业务层接口尽量不变。
- 计费账本需要可追溯、可审计。
- 任务执行应支持异步队列，尤其是视频生成。
- 生成链路保留幂等和重试能力。
- 敏感素材与品牌资料需做访问控制。

## Technology Direction
- 业务后端以 Java 为主，承载租户、项目、权限、计费、素材、审计、管理后台接口等核心业务域。
- 模型相关能力以 Python 为主，承载模型供应商适配、提示词组装、品牌约束注入、异步生成任务编排等 AI 相关逻辑。
- Java 与 Python 之间通过清晰的内部 API、消息队列或任务协议通信，避免 Python 直接承担大部分业务主域。
- 平台设计应允许后续把 Python 模型层独立扩缩容，而不影响 Java 业务服务。

## Suggested Architecture
- `auth-service` (Java): 身份、租户、RBAC。
- `billing-service` (Java): 钱包、额度、价格、账本。
- `asset-service` (Java): 素材库、品牌知识、文件元数据。
- `admin-api` (Java): 超管、租户、项目、模型配置、审计等业务管理接口。
- `model-gateway` (Python): 统一模型适配层与供应商切换。
- `generation-worker` (Python): 任务编排、提示词组装、品牌约束注入、结果回传。
- `admin-console`: 超管后台。
- `tenant-console`: 租户/项目/用户控制台。

## Open Questions
- 租户充值是否需要真实支付渠道，还是先做后台人工充值。
- 品牌知识引用是手选素材优先，还是允许系统自动召回。
- 视频生成是否只做异步任务编排，还是需要完整工作流审批。
- 是否需要项目级审批流来控制高成本模型调用。
- 是否需要配额预警、余额预警和自动停用策略。
- 是否需要内容审核供应商接入。

## V1 Completion Criteria
- 能创建租户、项目、成员并完成权限隔离。
- 能接入至少两家不同模型供应商并通过统一接口调用。
- 能完成文案、图片、视频三类任务的基础编排。
- 能配置模型可用范围、平台价格和租户配额。
- 能基于品牌资料和素材生成结果，而不是纯裸 prompt。
- 能看到清晰的消耗、账本、审计和任务记录。
