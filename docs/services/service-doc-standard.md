# 服务文档编写规范

## 目的

每个服务都必须有独立的中文说明文档，便于后续你自己排查问题、调整配置、接手维护。

## 适用范围

当前仓库里的所有服务都适用：

- Java 微服务
- Python 模型服务
- 前端管理台服务
- 后续新增的任何网关、中台、异步任务、模型代理服务

## 文档最少包含的内容

每个服务文档至少要写清楚这些部分：

1. 服务定位  
   说明这个服务负责什么，不负责什么。
2. 当前职责  
   说明当前已经落地的业务能力。
3. 后续职责  
   说明这个服务后面还会承接什么能力。
4. 端口与服务名  
   说明本地端口、`spring.application.name` 或服务标识。
5. 主要接口  
   写出关键 API 或入口。
6. 数据存储  
   说明这个服务当前依赖哪些表、哪些数据库。
7. 配置项  
   说明关键环境变量和配置参数。
8. 启动方式  
   写出本地启动命令和脚本位置。
9. 日志与追踪  
   说明 `X-Trace-Id`、日志文件位置、排查入口。
10. 当前已知限制  
   说明还没做完的地方和临时方案。

## 强制要求

- 文档语言统一用中文
- 后续新增服务时，服务代码和服务文档必须同一次提交完成
- 如果服务职责有明显变化，对应文档必须同步更新
- 如果服务接入数据库、网关、注册中心、消息队列，也必须把依赖关系写进文档

## 推荐目录

服务文档统一放在：

`docs/services/`
